Network class

A pure-NumPy implementation of a 2-hidden-layer fully-connected neural network (MLP). Inside the DQN agent it maps environment states → Q-values for every possible action, with no external ML framework needed.

Architecture Diagram

Input: (B, state_size) Hidden: ReLU non-linearity Output: (B, action_size) Q-values Weights: He-initialised

__init__ — Constructor

Called once when you write net = Network(state_size, action_size). Its sole job is to create and He-initialise all 6 learnable parameter arrays.

Full Annotated Code

def __init__(self, state_size: int, action_size: int, hidden_size: int = 64): rng = np.random.default_rng() # ── Layer 1 ────────────────────────────── self.w1 = rng.normal(0, np.sqrt(2.0 / state_size), size=(hidden_size, state_size)) self.b1 = np.zeros((hidden_size,)) # ── Layer 2 ────────────────────────────── self.w2 = rng.normal(0, np.sqrt(2.0 / hidden_size), size=(hidden_size, hidden_size)) self.b2 = np.zeros((hidden_size,)) # ── Output Layer ───────────────────────── self.w3 = rng.normal(0, np.sqrt(2.0 / hidden_size), size=(action_size, hidden_size)) self.b3 = np.zeros((action_size,))

Parameters Explained

ParamTypeMeaning
state_sizeintDimensionality of the env observation (e.g. 4 for CartPole)
action_sizeintNumber of discrete actions the agent can choose
hidden_sizeint = 64Width of each hidden layer; default 64 is a solid starting point

Shapes Created

  • w1 (hidden_size, state_size) — maps input → hidden 1
  • b1 (hidden_size,) — bias for each hidden-1 neuron, init 0
  • w2 (hidden_size, hidden_size) — hidden 1 → hidden 2 (square)
  • b2 (hidden_size,) — bias for hidden 2
  • w3 (action_size, hidden_size) — hidden 2 → Q output
  • b3 (action_size,) — one bias per action

Line-by-line Breakdown

  • rng = np.random.default_rng() Creates a modern NumPy Generator object (introduced in NumPy 1.17). It's faster and statistically better than the legacy np.random.randn. Seeding is optional — omitting the seed means results vary each run, which is fine for RL experiments.
  • rng.normal(0, std, size=(rows, cols)) Draws a matrix of random floats from a Normal distribution with mean=0 and the given standard deviation. The size argument controls the output shape directly — no separate reshape needed.
  • np.sqrt(2.0 / fan_in) This is the He / Kaiming initialisation formula. fan_in is the number of incoming connections to each neuron. For w1, fan_in = state_size; for w2 and w3, fan_in = hidden_size. The factor of 2 accounts for ReLU cutting activations roughly in half.
  • np.zeros((n,)) Allocates a 1-D array of n zeros as float64. Biases start at zero — unlike weights, they don't suffer from symmetry-breaking issues so random initialisation is unnecessary.
  • self.wX / self.bX Assigning to self stores the arrays as instance attributes — they persist for the lifetime of the object and are accessible from every method via self.

Why He Initialisation?

std = √(2 / fan_in) → W ~ 𝒩(0, std²)
  • Problem with small fixed std If weights are initialised too small (e.g., std=0.01), signals shrink layer by layer → vanishing gradients. Too large → exploding gradients.
  • Why ×2? ReLU sets ~50% of activations to 0 (the negative half). This halves the effective variance. Multiplying the std formula by √2 exactly compensates, keeping variance constant across depth.
  • Result Activations and gradients maintain roughly unit variance at initialisation, allowing training to start immediately without exploding or vanishing.

forward() — Forward Pass

Takes a batch of states and produces one Q-value per action per state. Pure vectorised matrix math — no Python loops, no autograd.

Full Annotated Code

def forward(self, x: np.ndarray) -> np.ndarray: """x : state batch → shape (B, state_size) returns Q-values → shape (B, action_size)""" z1 = x @ self.w1.T + self.b1 # (B, H) linear pre-activation layer 1 a1 = np.maximum(0.0, z1) # (B, H) ReLU — kills negatives z2 = a1 @ self.w2.T + self.b2 # (B, H) linear pre-activation layer 2 a2 = np.maximum(0.0, z2) # (B, H) ReLU q = a2 @ self.w3.T + self.b3 # (B, A) raw Q-values — no activation! return q

Step-by-step Flow

1

Linear transform — Layer 1 (B, S) → (B, H)

z1 = x @ w1ᵀ + b1

@ is Python's matrix-multiply operator (PEP 465). w1 is stored as (H, S); transposing it to (S, H) makes shapes align: (B,S)×(S,H) = (B,H). NumPy broadcasts b1 of shape (H,) across all B rows automatically — no explicit reshape needed.

2

ReLU activation — Layer 1 (B, H) → (B, H)

a1 = max(0, z1) — elementwise

np.maximum(0.0, z1) replaces every negative value with 0, preserving positives unchanged. This is the ReLU (Rectified Linear Unit) — it introduces the non-linearity that lets the network approximate any function. Without it, stacking linear layers would still be linear.

3

Linear transform — Layer 2 (B, H) → (B, H)

z2 = a1 @ w2ᵀ + b2

Same pattern as step 1, but between the two hidden layers. Because both input and output are size H, w2 is a square matrix (H, H).

4

ReLU activation — Layer 2

a2 = max(0, z2)

Second non-linearity, identical in behaviour to step 2.

5

Output Q-values — NO activation (B, H) → (B, A)

q = a2 @ w3ᵀ + b3

No ReLU here! Q-values can be negative (representing bad expected total reward). q[i, a] is the estimated cumulative reward for taking action a in state x[i]. The agent picks argmax(q[i]) to act greedily.

Python Syntax Breakdown

  • @ (matmul) Matrix multiplication operator introduced in Python 3.5 (PEP 465). A @ B is equivalent to np.matmul(A, B). Cleaner than np.dot for 2-D arrays and reads naturally as math notation.
  • .T (transpose) Read-only property on any NumPy array. For 2-D arrays it swaps axes: (H, S) → (S, H). No data is copied — it's a view. We need it because weights are stored in output-major order (out, in) but the matrix multiply needs them as (in, out).
  • np.maximum(a, b) Elementwise max — picks the larger value at each position. Not to be confused with np.max(arr, axis=…) which reduces along an axis. Here we compare every element against the scalar 0.
  • Broadcasting When you add b1 of shape (H,) to z1 of shape (B, H), NumPy automatically replicates b1 B times along the batch axis. No explicit reshape or tile is needed.
  • Type hints x: np.ndarray and -> np.ndarray are documentation-only annotations. Python does not enforce them at runtime — they exist purely for readability and IDE support.

copy_from() — Target Network Sync

Overwrites every parameter of self with an independent (deep) copy of other's parameters. This is how the DQN target network is periodically synced with the policy network.

Full Annotated Code

def copy_from(self, other: "Network") -> None: """Copy parameters from another network (used for target-net sync).""" self.w1 = other.w1.copy() self.b1 = other.b1.copy() self.w2 = other.w2.copy() self.b2 = other.b2.copy() self.w3 = other.w3.copy() self.b3 = other.b3.copy()

Role in DQN Architecture

The target network provides stable TD targets. If you updated the target every step it would chase itself and training would diverge. Instead it's frozen for 500 steps and then snapped to the policy network via copy_from.

Python Syntax Breakdown

  • Why .copy()? Without .copy(), writing self.w1 = other.w1 makes both networks share the exact same array object in memory. Any gradient update would silently modify both networks at once — completely breaking the target net's role as a stable reference.
  • ndarray.copy() Allocates a brand-new array of the same shape and dtype, then copies all values. The two networks are now fully independent in memory.
  • Forward reference "Network" The type hint other: "Network" wraps the class name in quotes. Python hasn't finished defining Network yet when it parses the method signatures (we're still inside the class body), so a bare Network would raise a NameError. Quoting defers resolution. From Python 3.10+ you can use from __future__ import annotations instead (which this file already does at line 1).
  • -> None Return type annotation meaning the method returns nothing. The operation is purely a side-effect — it mutates self in-place.

parameters() — Parameter Accessor

Returns all 6 learnable arrays as one Python tuple. This is the standard interface the Adam optimiser uses to iterate over and update every parameter.

Full Annotated Code

def parameters(self) -> Tuple[np.ndarray, ...]: return (self.w1, self.b1, self.w2, self.b2, self.w3, self.b3)

Python Syntax Breakdown

  • Tuple return Returning multiple values separated by commas is Python sugar for returning a tuple. return a, b, c is identical to return (a, b, c). The caller can unpack: w1, b1, w2, b2, w3, b3 = net.parameters().
  • Tuple[np.ndarray, ...] Type hint from the typing module (imported at line 13). The ... (Ellipsis literal) inside Tuple[…] means: a homogeneous tuple of any length whose elements are all np.ndarray. This is the standard way to annotate variable-length tuples.
  • No copy — live references The returned arrays are the same objects as self.w1 etc., not copies. This is intentional: the optimiser does param -= lr * update which modifies the array in-place, and those changes are immediately reflected in the network's own attributes.
  • Order matters The order matches _param_names = ("w1","b1","w2","b2","w3","b3") in DQNAgent.__init__. The zip pairs names with arrays positionally, so changing the order here would silently assign wrong Adam moment arrays to wrong parameters.

How It's Used in DQNAgent

_param_names = ("w1", "b1", "w2", "b2", "w3", "b3") # Build Adam moment dicts keyed by parameter name self._adam_m = {n: np.zeros_like(p) for n, p in zip(_param_names, self.policy_net.parameters())} # Later, during training, each parameter is updated: for name, param, grad in param_grad_pairs: m_hat = ... # bias-corrected 1st moment v_hat = ... # bias-corrected 2nd moment param -= lr * m_hat / (np.sqrt(v_hat) + eps)

Because param holds a reference to the same array as self.policy_net.w1 (etc.), the in-place -= directly updates the network's weights with no extra copy.
